# PICO — Music to World

> Sing into a PICO device. Get a live music sheet and a navigable 3D world.

PICO is a creative AI pipeline that turns a user's voice into two outputs simultaneously: a live-drawn music sheet transcribed from their melody, and a photorealistic 3D world generated by World Labs Marble — shaped by the emotional feel of the music.

---

## How it works

```
User sings (PICO mic)
       │
       ▼
  Claude (AI Model 1)
  ─ transcribes melody
  ─ detects BPM, key, mood
       │
       ├──► Sheet music (drawn live on PICO display)
       │
       ▼
  Claude (AI Model 2)
  ─ takes music analysis + user Q&A
  ─ writes a cinematic world prompt
       │
       ▼
  World Labs Marble API
  ─ generates navigable 3D world
       │
       ▼
  Unity (PICO app)
  ─ renders world via Gaussian splats
  ─ user explores their world in VR
```

The user is asked a short set of questions (vibe, setting, time of day, season, existing song reference) that Claude uses alongside the music analysis to craft the world prompt.

## Stack

| Layer | Technology |
|---|---|
| Input device | PICO XR headset (mic + display) |
| Music analysis | Claude API (`claude-sonnet-4`) |
| Sheet music rendering | HTML5 Canvas |
| World prompt generation | Claude API (`claude-sonnet-4`) |
| 3D world generation | World Labs Marble API |
| World rendering (web) | World Labs Spark / Three.js |
| World rendering (device) | Unity 6.0 + aras-p Gaussian splat plugin |
| Export formats | `.spz` / `.ply` (visuals) + `.glb` (collider mesh) |

## Getting started

### Prerequisites

- Anthropic API key
- World Labs API key
- A browser with microphone access (for the prototype)
- Unity 6.0 + aras-p plugin (for PICO device deployment)

### Run the prototype

Open `pico-pipeline.html` in a browser. No build step required.

1. Enter your Anthropic and World Labs API keys
2. Click the mic button and sing or hum a melody
3. Stop recording, fill in the world settings (vibe, environment, etc.)
4. Click **Generate Sheet + World**

Claude will analyze your melody, draw the sheet music, generate a world prompt, and kick off world generation via the Marble API (~5 min). The world embeds directly in the browser when ready.

---

## World Labs → Unity

Marble exports two things:

- **`.spz` / `.ply`** — Gaussian splat file (the visual world). Import via the [aras-p Unity plugin](https://github.com/aras-p/UnityGaussianSplatting). Use the 2M splat version for full quality.
- **`.glb`** — collider mesh for physics. Import as a standard Unity asset; hide the renderer and keep the collider active.

> **Note:** Use Unity 6.0 specifically (`6000.0.23f1`). The aras-p plugin does not support VR on Unity 6.3. URP is recommended. Enable HDR on your URP asset to avoid black screens in VR.

Current limitation: Marble exports static worlds. Animated elements do not transfer.

## User questions

The questionnaire that shapes the world prompt:

| Question | Options |
|---|---|
| Vibe | Dreamy, dark, energetic, nostalgic, epic, peaceful |
| Setting | Forest, urban, ocean, mountains, fantasy, desert, space |
| Time of day | Golden hour, night, dawn, midday, overcast |
| Season | Spring, summer, autumn, winter |
| Existing song? | Free text (optional) |

## Notes

- If Claude cannot process the audio directly, it falls back to generating a plausible melody from the lyrics field.
- The World Labs API is asynchronous. World generation takes ~5 minutes. The prototype polls every 5 seconds and embeds the result when complete.
- Sheet music is rendered as a simple single-staff treble clef melody. Bar lines, beam groups, and time signatures are on the roadmap.
